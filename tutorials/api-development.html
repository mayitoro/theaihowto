Build Your Own App with the OpenAI and Claude APIs

A complete developer guide to building AI-powered applications. Covers API authentication, chat completions, token management, streaming, and deployment to production.

📖
What you'll learn: A complete developer guide to building AI-powered applications. Covers API authentication, chat completions, token management, streaming, and deployment to production.
Introduction

Building your own AI-powered application gives you unlimited control over the user experience, pricing, and capabilities. This guide walks through everything a developer needs to go from zero to a production-ready AI app using the OpenAI and Anthropic APIs.

Step-by-step guide

Follow these steps

1

Get your API keys

For OpenAI: Go to platform.openai.com → API Keys → Create new secret key. For Anthropic: Go to console.anthropic.com → API Keys → Create Key. Store these securely — never commit them to git.

# Install SDKs pip install openai anthropic # Or with npm npm install openai @anthropic-ai/sdk
2

Your first API call

The simplest possible API call — a single question and answer:

from openai import OpenAI client = OpenAI(api_key='your-key') response = client.chat.completions.create( model='gpt-4o', messages=[{ 'role': 'user', 'content': 'What is the capital of France?' }] ) print(response.choices[0].message.content)
3

Build a chat conversation

Maintain conversation history by passing all previous messages in each request:

messages = [ {'role': 'system', 'content': 'You are a helpful assistant.'}, ] while True: user_input = input('You: ') messages.append({'role': 'user', 'content': user_input}) response = client.chat.completions.create( model='gpt-4o', messages=messages ) reply = response.choices[0].message.content messages.append({'role': 'assistant', 'content': reply}) print(f'AI: {reply}')
4

Implement streaming

Stream responses word by word for a better user experience:

stream = client.chat.completions.create( model='gpt-4o', messages=messages, stream=True ) for chunk in stream: if chunk.choices[0].delta.content: print(chunk.choices[0].delta.content, end='', flush=True)
5

Manage tokens & costs

Monitor token usage to control costs:

# After each API call usage = response.usage print(f'Prompt tokens: {usage.prompt_tokens}') print(f'Response tokens: {usage.completion_tokens}') print(f'Total: {usage.total_tokens}') # GPT-4o pricing (May 2026) # Input: $5.00 per 1M tokens # Output: $15.00 per 1M tokens
6

Deploy to production

Use environment variables, never hardcode API keys. For deployment, use Vercel (Node.js/Python), Railway, or Render. Always keep API calls server-side to protect your keys from being exposed in client code.

# .env file (never commit this) OPENAI_API_KEY=sk-... ANTHROPIC_API_KEY=sk-ant-... # Load in code import os from dotenv import load_dotenv load_dotenv() client = OpenAI(api_key=os.getenv('OPENAI_API_KEY'))
Pro tips

Do this

  • ✓Always use environment variables — never hardcode API keys in source code
  • ✓Set max_tokens to avoid unexpectedly long (expensive) responses
  • ✓Use a system prompt to define AI behavior consistently across all conversations
  • ✓Cache common responses to reduce API costs and improve response times
Common mistakes

Avoid these

  • ✗Exposing API keys in client-side JavaScript — always keep API calls server-side
  • ✗Not handling API errors — implement retry logic with exponential backoff
  • ✗Forgetting to limit context window — very long conversations get expensive fast
  • ✗Not streaming responses — users expect to see text appear progressively
